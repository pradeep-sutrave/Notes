# 📘 UNIT – 12 : ERROR HANDLING & SECURITY IN PHP (DETAILED)

This unit teaches PHP to:

* Handle mistakes safely
* Protect applications from attacks
* Build **secure, production-ready code**

---

## 1️⃣ What is an Error?

### Definition (Exam-ready ✍️)

> An error is a problem that occurs during program execution and disrupts the normal flow of the program.

Errors are **normal**, but **handling them properly** is professional coding.

---

## 2️⃣ Types of Errors in PHP

### 🔹 1. Parse Error

* Syntax mistake

```php
echo "Hello
```

---

### 🔹 2. Fatal Error

* Script stops execution

```php
undefinedFunction();
```

---

### 🔹 3. Warning

* Script continues but shows warning

```php
include("file.txt");
```

---

### 🔹 4. Notice

* Minor issue

```php
echo $x;
```

---

## 3️⃣ Displaying Errors (Development Mode)

```php
<?php
error_reporting(E_ALL);
ini_set("display_errors", 1);
?>
```

✔ Use in **development**
❌ Disable in **production**

---

## 4️⃣ Hiding Errors (Production Mode)

```php
<?php
ini_set("display_errors", 0);
?>
```

✔ Prevents exposing sensitive data

---

## 5️⃣ Custom Error Handling

### Using `set_error_handler()`

```php
<?php
function myError($errno, $errstr) {
    echo "Error: [$errno] $errstr";
}

set_error_handler("myError");
echo $x;
?>
```

✔ Better control
✔ Clean error messages

---

## 6️⃣ Exception Handling (`try–catch`)

Used for **runtime error handling**.

```php
<?php
try {
    if (!file_exists("data.txt")) {
        throw new Exception("File not found");
    }
} catch (Exception $e) {
    echo $e->getMessage();
}
?>
```

✔ Prevents script crash
✔ Used in database & file handling

### 🔥 3. CSRF

---

## 9️⃣ Preventing SQL Injection (MOST IMPORTANT 🔥)

### ❌ Insecure Code

```php
$sql = "SELECT * FROM users WHERE user='$u' AND pass='$p'";
```

---

### ✅ Secure Code (Prepared Statements)

```php
$stmt = mysqli_prepare($conn,
    "SELECT * FROM users WHERE user=? AND pass=?");
mysqli_stmt_bind_param($stmt, "ss", $u, $p);
mysqli_stmt_execute($stmt);
```

✔ Stops injection completely

---

## 🔟 Preventing XSS Attacks

### Use `htmlspecialchars()`

```php
<?php
$name = htmlspecialchars($_POST["name"]);
echo $name;
?>
```

✔ Converts HTML to text
✔ Blocks scripts

---

## 1️⃣1️⃣ Input Validation & Sanitization

```php
<?php
$email = filter_input(INPUT_POST, "email",
         FILTER_VALIDATE_EMAIL);

if (!$email) {
    echo "Invalid email";
}
?>
```

✔ Accept only valid data
✔ Reject bad input

---

## 1️⃣2️⃣ Password Security (CRITICAL 🔐)

### ❌ Never Do This

```php
$password = "1234";
```

---

### ✅ Correct Way

```php
$hash = password_hash("1234", PASSWORD_DEFAULT);
```

Verify:

```php
password_verify("1234", $hash);
```

✔ Strong hashing
✔ Industry standard

---

## 1️⃣3️⃣ Session Security

```php
session_start();
session_regenerate_id(true);
```

✔ Prevents session hijacking
✔ Must be done after login

---

## 1️⃣4️⃣ File Upload Security

```php
$allowed = ["jpg", "png"];
$ext = pathinfo($_FILES["file"]["name"], PATHINFO_EXTENSION);

if (!in_array($ext, $allowed)) {
    die("Invalid file type");
}
```

✔ Prevents malware upload
