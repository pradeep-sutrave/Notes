# 📘 UNIT – 13 : OBJECT ORIENTED PROGRAMMING (OOP) IN PHP (DETAILED)

This unit teaches how **real-world PHP applications** (Laravel, frameworks, large projects) are written.

---

## 1️⃣ What is OOP?

### Definition (Exam-ready ✍️)

> Object Oriented Programming is a programming paradigm that organizes code using **classes and objects** instead of functions only.

---

## 2️⃣ Why OOP in PHP?

Without OOP (procedural PHP):

* Code becomes messy
* Hard to maintain
* No reusability

With OOP:

* Clean structure
* Reusable code
* Secure & scalable
* Framework-ready (Laravel uses OOP)

## 4️⃣ Class & Object

### 🔹 Class

A **blueprint** for creating objects.

```php
<?php
class Student {
    public $name;
    public $branch;

    function show() {
        echo $this->name . " - " . $this->branch;
    }
}
?>
```

---

### 🔹 Object

An **instance** of a class.

```php
<?php
$s1 = new Student();
$s1->name = "Bhupathi";
$s1->branch = "CSE";

$s1->show();
?>
```

📌 `$this` refers to current object.

---

## 5️⃣ Constructor

Automatically runs when object is created.

```php
<?php
class Student {
    public $name;

    function __construct($name) {
        $this->name = $name;
    }

    function show() {
        echo $this->name;
    }
}

$s1 = new Student("Bhupathi");
$s1->show();
?>
```

---

## 6️⃣ Access Modifiers

Control visibility of class members.

| Modifier    | Access             |
| ----------- | ------------------ |
| `public`    | Anywhere           |
| `protected` | Same class + child |
| `private`   | Same class only    |

```php
class Demo {
    private $x = 10;
}
```

---

## 7️⃣ Encapsulation

Binding **data + methods** together and **restricting access**.

```php
<?php
class Account {
    private $balance = 1000;

    public function getBalance() {
        return $this->balance;
    }
}

$acc = new Account();
echo $acc->getBalance();
?>
```

✔ Data hiding
✔ Security

---

## 8️⃣ Inheritance

One class **inherits** another.

```php
<?php
class Person {
    public $name;
}

class Student extends Person {
    public $branch;
}

$s = new Student();
$s->name = "Bhupathi";
$s->branch = "CSE";
?>
```

✔ Code reuse
✔ Hierarchy

---

## 9️⃣ Method Overriding

Child class redefines parent method.

```php
<?php
class A {
    function show() {
        echo "Parent";
    }
}

class B extends A {
    function show() {
        echo "Child";
    }
}

$obj = new B();
$obj->show();
?>
```

---

## 🔟 Polymorphism

Same method name, **different behavior**.

```php
<?php
class Shape {
    function area() {}
}

class Square extends Shape {
    function area() {
        echo "Square Area";
    }
}

class Circle extends Shape {
    function area() {
        echo "Circle Area";
    }
}
?>
```

---

## 1️⃣1️⃣ Abstract Classes

Cannot create object of abstract class.

```php
<?php
abstract class Bank {
    abstract function rate();
}

class SBI extends Bank {
    function rate() {
        echo "7%";
    }
}

$b = new SBI();
$b->rate();
?>
```

---

## 1️⃣2️⃣ Interfaces

Defines **rules** that class must follow.

```php
<?php
interface Payment {
    public function pay();
}

class UPI implements Payment {
    public function pay() {
        echo "Payment via UPI";
    }
}
?>
```

📌 Interface supports **multiple inheritance**

---

## 1️⃣3️⃣ Final Keyword

Prevents inheritance or overriding.

```php
final class Demo {}
```

---

## 1️⃣4️⃣ Static Members

Shared across all objects.

```php
<?php
class Counter {
    public static $count = 0;

    static function inc() {
        self::$count++;
    }
}

Counter::inc();
echo Counter::$count;
?>
```

---

## 1️⃣5️⃣ Namespaces (Intro)

Used to avoid class name conflicts.

```php
<?php
namespace App;

class User {}
?>
```
